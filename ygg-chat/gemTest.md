# Local Testing Plan for Client-Side Tool Execution

To test the new Client-Side Tool Execution flow locally, we need to simulate the production environment where the "Cloud" server (Railway) and the "Local" server (Electron) are distinct entities.

## Architecture for Testing

We will run **two** separate instances of the server locally:

1.  **Cloud Mimic Server (Port 3002)**
    *   **Role**: Simulates the Railway server.
    *   **Mode**: `VITE_ENVIRONMENT=web`
    *   **Responsibility**: Handles LLM generation, Supabase sync, and decides when to trigger a `tool_call`.
    *   **URL**: `http://localhost:3002/api`

2.  **Local Tool Server (Port 3001)**
    *   **Role**: Simulates the embedded Electron server.
    *   **Mode**: `VITE_ENVIRONMENT=electron` (default)
    *   **Responsibility**: Executes tools via `wslBridge` upon request from the client.
    *   **URL**: `http://localhost:3001/api`

3.  **Electron Client**
    *   **Config**: Points to the **Cloud Mimic Server** for chat messages.
    *   **Logic**: Receives `tool_call` from Port 3002, executes on Port 3001, sends results back to Port 3002.

## Step-by-Step Setup

### 1. Configure the "Cloud Mimic" Server

We need to ensure the server running on port 3002 behaves exactly like the production web server.

**Action**:
*   You will need a `.env` file in `server/` with your Supabase and OpenRouter credentials.
*   You can copy your existing `.env` to `.env.web` for clarity, or just use the existing one if it has the keys.

**Command to Start Cloud Mimic**:
Open a terminal in `server/`:
```bash
# Linux/WSL
PORT=3002 VITE_ENVIRONMENT=web npm run dev

# Windows (PowerShell)
$env:PORT="3002"; $env:VITE_ENVIRONMENT="web"; npm run dev
```

### 2. Configure the "Local Tool" Server

This is the standard local server that usually runs with Electron.

**Command to Start Local Server**:
Open a **second** terminal in `server/`:
```bash
# Linux/WSL (Default port is 3001)
VITE_ENVIRONMENT=electron npm run dev
```

### 3. Configure the Electron Client

We need to tell the Electron app to send chat messages to our "Cloud Mimic" (3002) instead of the real Railway URL.

**Action**:
*   Edit `client/ygg-chat-r/.env`:
    ```properties
    # Point to the Cloud Mimic
    VITE_API_URL=http://localhost:3002/api
    
    # Ensure Supabase keys are present for Authentication to work
    VITE_SUPABASE_URL=...
    VITE_SUPABASE_ANON_KEY=...
    
    # Ensure Environment is NOT web (so it acts as a client executioner)
    VITE_ENVIRONMENT=electron
    ```

**Command to Start Client**:
Open a **third** terminal in `client/ygg-chat-r/`:
```bash
npm run dev:electron
```

## Testing the Flow

1.  **Login**: Launch the Electron app. If not logged in, log in.
    *   *Note: Since both Client and Cloud Mimic use the same real Supabase project, the Auth Token generated by the Client will be valid on the Cloud Mimic server.*
2.  **Send a Message**: Type a prompt that requires a tool, e.g., "Read the file package.json in the current directory."
3.  **Observe Logs**:
    *   **Terminal 1 (Cloud Mimic - 3002)**: Should show the request arriving. It should generate a `tool_call` event and log "Client-side execution mode: halting...".
    *   **Client Console (Electron DevTools)**: Should show "Received tool_call", then "Executing local tool...".
    *   **Terminal 2 (Local Server - 3001)**: Should show `POST /tools/execute` request and the tool output.
    *   **Terminal 1 (Cloud Mimic - 3002)**: Should receive the continuation request (`POST /messages`) containing the tool result, and then stream the final answer.

## Troubleshooting

*   **Auth Errors (401)**: If the Cloud Mimic rejects the token, ensure both Client and Server `.env` files are pointing to the *same* Supabase project ID.
*   **CORS Errors**: The Server includes `cors` middleware. Ensure `http://localhost:5173` (or whatever origin Electron uses in dev) is in the allowed list or that the regex allows it.
    *   *Tip: In `server/src/index.ts`, the CORS config allows localhost. It should be fine.*
*   **Port Conflicts**: Ensure nothing else is using 3001 or 3002.
